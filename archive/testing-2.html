Tab Management Redesign

Cross-platform tab redesign with grouping functionality

Impact

1.8% Day 1 retention improvement across iOS and Android (1.2% iOS, 0.6% Android)

Shipped complete grouping on iOS in 6 months, with Android UI foundations laid before company shifted to premium features.

The Problem

Power users with 20+ tabs couldn't find what they needed. Chrome and Safari offered grouping. We didn't. This table-stakes gap was hurting retention.

Discovery & Strategy

Led end-to-end design across iOS and Android as part of a 2-person design team at Aloha Browser (4M users). Conducted competitive research analyzing Chrome, Safari, Edge, and Firefox tab grouping implementations. Identified key patterns: visual grouping, drag-and-drop organization, and persistent group state. Worked closely with product and engineering to define MVP scope that balanced user value with technical feasibility.

Solution

Flexible Grouping

Drag-and-drop interaction for quick tab organization

Designed two interaction paths: batch selection for power users organizing many tabs, and drag-and-drop for quick one-off organization.

Design Approach

Created a dual-mode system that serves different user needs. Power users can select multiple tabs simultaneously using checkboxes, then group them with a single actionâ€”essential when organizing 20+ tabs. Casual users can drag individual tabs between groups for spontaneous organization. Both interactions feel native to their respective platforms while maintaining the same underlying functionality.

Designed groups to be collapsible with clear visual hierarchy. Group headers show tab count and support quick actions like rename, color picker, and delete. Collaborated with iOS and Android engineers to ensure gestures felt platform-appropriateâ€”swipe actions on iOS, long-press on Android.

Mode Switcher

Unconventional mode switcher integrating Normal, Private, and Sync tabs

Created an unconventional tab mode switcher letting users toggle between Normal, Private, and Sync modes without leaving the tab manager.

Design Rationale

Most browsers force users to exit the tab manager and navigate through menus to switch between normal and private modes. This creates friction when users want to quickly check private tabs or move between contexts. By embedding the mode switcher directly in the tab manager, we reduced navigation steps and made mode-switching a first-class action.

The design uses clear visual differentiationâ€”subtle background color changes and iconsâ€”to communicate the current mode. Sync mode (unique to Aloha) gets equal prominence, encouraging cross-device tab management. Validated this approach with engineering early to ensure technical feasibility across iOS and Android.

Platform-Specific Design

iOS and Android implementations respecting platform conventions

Respected iOS and Android conventions rather than forcing identical experiences. iOS shipped with complete grouping; Android received UI improvements with grouping planned for future release.

Cross-Platform Strategy

Shipped iOS first to validate the grouping concept with real user metrics before investing in full Android implementation. This de-risked the feature and informed Android design decisions based on actual usage patterns. iOS users adopted grouping quicklyâ€”43% created at least one group within the first week.

For Android, partnered closely with the Android team to design UI foundations that matched Material Design patterns while maintaining feature parity with iOS. Iterated post-launch when initial tab card shapes caused visual issues across landscape and portrait orientations. Final Android design simplified the visual system while improving performance.

Android Performance Solution

List view toggle optimizing performance for older devices

Turned a constraint into a feature: designed a toggle between rich tab previews and an efficient list mode for older devices.

Technical Constraints & Solutions

Android engineering flagged performance issues with rich tab previews on devices with limited RAM. Rather than compromise the experience for all users, designed a user-controlled toggle between preview mode and list mode. List mode uses text-only representation, reducing memory overhead by 60% while maintaining full grouping functionality.

Positioned the toggle as a power user feature rather than a fallback. Users with newer devices still switched to list mode when managing 50+ tabs, discovering it was faster for scanning large tab collections. This turned a technical constraint into a legitimate feature that served different user needs and contexts.
